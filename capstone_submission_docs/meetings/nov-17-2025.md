#### **1.1. Submission System Architecture**

The team outlined the foundational components for the submission feature:

- **Submission Lifecycle:** Define the state machine for submissions (draft → submitted → graded → published)
- **Workspace Integration:** Track when workspaces are created, edited, and finalized for submission
- **Metadata Storage:** Store submission timestamps, last edited timestamps, and user references

#### **1.2. Event Tracking Requirements**

Key data points to be tracked for each submission:

- **Submission Creation Time:** When the student first submits their work
- **Last Edited Timestamp:** Granular tracking of when code was last modified before submission
- **Edit History:** Maintain the event log for potential instructor review
- **Workspace State:** Capture the final state of the workspace at submission time

#### **1.3. Backend Infrastructure**

The team identified necessary backend components:

- **Submission Schema:** Design database model with fields for user ID, assignment ID, status, timestamps
- **Terraform Setup:** Configure cloud infrastructure for submission processing
- **Storage Helpers:** Implement utility functions for consistent submission data handling
- **API Endpoints:** Create mutations for submitting assignments and querying submission status

#### **1.4. Performance Considerations**

- Ensure workspace queries remain fast even with large edit histories
- Optimize submission status checks for real-time UI updates
- Plan for eventual scaling to thousands of concurrent submissions

### **2. Decisions Made**

1. **Submission Status Model:** Implement as a finite state machine with states: `draft`, `submitted`, `graded`, `published`, `hidden`
2. **Timestamp Strategy:** Store both submission time and last edited time for assignment deadline analysis
3. **Sidecar Architecture:** Use serverless function approach for submission processing (via Terraform)
4. **Event Persistence:** Maintain complete edit history in database for potential plagiarism detection
5. **Sprint Focus:** Prioritize backend schema and mutations; frontend UI deferred to Sprint 9
