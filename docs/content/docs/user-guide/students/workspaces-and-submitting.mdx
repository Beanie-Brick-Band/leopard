# Workspaces and Submitting

## Launching a Workspace

The workspace is your coding environment for the assignment.

To launch it:

1. Open the assignment.
2. Click `Launch Workspace`.
3. Wait while Leopard prepares it.
4. Click `Open Workspace` when it is ready.

You may see messages like:

- `Launching workspace`
- `Workspace ready`

## Working in the Workspace

Inside the workspace, complete the assignment like you would in a normal coding environment.

Depending on the course setup, you may:

- edit files
- run code
- test your solution
- work from starter code

## Starter Code

Some assignments include starter code from the teacher.

Starter code may contain:

- a project skeleton
- partially completed code
- instructions
- sample files

If starter code exists, use it unless your teacher tells you otherwise.

## Submitting Your Work

Leopard ties submissions to the assignment workspace.

To submit:

1. Open the assignment page.
2. Make sure you have already launched the workspace for that assignment.
3. Click `Submit Assignment`.
4. Confirm the submission if prompted.

After submitting, Leopard records:

- that the assignment was submitted
- the submission time
- the workspace attached to the submission

## Important Submission Rules

### You must launch the workspace first

If you have not launched the workspace for that assignment yet, Leopard will not let you submit.

### Due dates matter

If the due date has passed, normal submission is blocked.

Because of that, do not wait until the last minute to test your code or launch the workspace for the first time.

### Submission is final

After you submit, Leopard currently treats that submission as final in the student UI.

Because of that:

- review your work before submitting
- make sure you are in the correct assignment workspace
- use the preview in the confirmation dialog if available

## Student Tips

- Launch the workspace early.
- Make sure you are working in the correct assignment workspace.
- After submitting, return to the assignment page and check the status panel.
