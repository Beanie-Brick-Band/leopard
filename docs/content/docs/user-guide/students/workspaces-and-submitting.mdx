# Workspaces and Submitting

Leopard submits the workspace attached to the assignment you are viewing, so the safest flow is: open the assignment, launch the workspace, do your work, then return and submit from the same page.

1. Open the assignment.
2. Read the description so you know which task you are opening.
3. Click `Launch Workspace`.
4. Wait while Leopard prepares the environment.
5. Return to the assignment page when you are ready to submit from Leopard.

## Working in the Workspace

Inside the workspace, complete the assignment like you would in a normal coding environment.

Depending on the course setup, you may:

- edit files
- run code
- test your solution
- work from starter code

## Starter Code

If the teacher attached starter code, Leopard uses that starter project as the starting point for your workspace.

1. Open the assignment page.
2. Confirm that `Submit Assignment` is available.
3. Click `Submit Assignment`.
4. Review the `Current Work` snapshot in the confirmation dialog.
5. Click `Confirm Submit`.

Leopard records the submission time and links that submission to the workspace for this assignment.

## Important Submission Rules

- If `Submit Assignment` is disabled, launch the workspace first.
- Leopard submits the workspace tied to the assignment currently on screen.
- Due dates still control whether normal submission is allowed.
- The current student UI treats submission as final, so use the confirmation dialog to double-check before you finish.

## Student Tips

- Launch the workspace early instead of for the first time right before the deadline.
- Make sure you are on the correct assignment page before you submit.
- After submitting, reopen the assignment page and verify the status panel changed to `Submitted`.
