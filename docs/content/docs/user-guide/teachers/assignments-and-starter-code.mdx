# Assignments and Starter Code

## Creating an Assignment

To create a new assignment:

1. Open the classroom.
2. Choose the option to create an assignment.
3. Fill out the form.

The form includes:

- assignment name
- release date
- due date
- description
- optional starter code

## Writing Assignment Instructions

Students rely on the assignment page as their main source of truth.

A good assignment description usually explains:

- what students need to build
- where they should begin
- which files matter most
- how the work will be graded
- what counts as complete

## Release Date and Due Date

When creating or editing an assignment, you will see:

- `Release Date`
- `Due Date`

The due date is especially important because Leopard uses it when deciding whether students can still submit.

## Starter Code

Leopard lets teachers attach starter code to an assignment as a ZIP file.

This is useful when you want students to begin with:

- a prepared project
- a template
- partially completed code
- a controlled file structure

### Uploading Starter Code

During assignment creation:

1. Use the starter code uploader.
2. Choose a ZIP file.
3. Complete the upload.
4. Finish creating the assignment.

After the assignment already exists:

1. Open the assignment.
2. Find the `Starter Code` section.
3. Upload a ZIP file there.

### Replacing or Removing Starter Code

You can also:

- upload new starter code
- replace older starter code
- remove starter code entirely

Be careful when changing starter code after students have already started working.

## Editing an Assignment

You can usually edit:

- assignment name
- release date
- due date
- description

This is useful when:

- the deadline changed
- instructions need clarification
- there was a typo
- the assignment wording needs improvement

## Deleting an Assignment

Deleting an assignment is a major action.

Use it only when:

- the assignment was created by mistake
- the course plan changed
- you no longer need its submission records in active use

## Tips

- Write assignment instructions as if students will only read that one page.
- Set due dates with a buffer.
- Upload starter code only when it helps.
