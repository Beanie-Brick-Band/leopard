# Assignments and Starter Code

The current assignment creation flow lives inside each classroom and uses one form for the name, schedule, description, and optional starter ZIP.

1. Open the classroom.
2. Click `New Assignment`.
3. Fill in `Name`.
4. Set `Release Date` and `Due Date`.
5. Write the instructions in `Description`.
6. Upload optional starter code.
7. Click `Create Assignment`.

## Writing Assignment Instructions

Students rely on the assignment page as their main source of truth.

A good assignment description usually explains:

- what students need to build
- where they should begin
- which files matter most
- how the work will be graded
- what counts as complete

## Release Date and Due Date

Leopard uses these dates directly:

- `Release Date` controls when the assignment should become available.
- `Due Date` controls the deadline Leopard uses for student submission.
- The due date must be after the release date.

## Starter Code

Leopard accepts starter code as a ZIP file.

This is useful when you want students to begin with:

- a prepared project
- a template
- partially completed code
- a controlled file structure

### Uploading Starter Code

- Upload the ZIP during assignment creation if students should start from a template immediately.
- After the assignment exists, the assignment page still includes a `Starter Code` section for updates.

### Replacing or Removing Starter Code

- You can upload a new ZIP to replace the old one.
- You can remove starter code entirely from the assignment page.
- Be careful when changing starter code after students have already started working.

## Editing an Assignment

- The assignment detail page lets you edit the name, release date, due date, and description later.
- This is the right place to fix deadline changes, typos, or missing clarification.

## Deleting an Assignment

Deleting an assignment is a major action.

Use it only when:

- the assignment was created by mistake
- the course plan changed
- you no longer need its submission records in active use

## Tips

- Write assignment instructions as if students will only read that one page.
- Set due dates with a buffer.
- Upload starter code only when it helps students start faster or more consistently.
