# Review, Grading, and Publishing

Teachers grade from two connected screens:

- the assignment page, which lists submissions and statistics
- the review page, which shows replay and grading controls for one submission

## Viewing Submissions

Open a classroom, open an assignment, and use the `Submissions` table to jump into individual student work.

This page combines the main grading tools in one place:

- `Submission Details` confirms which student and workspace you are reviewing.
- `Replay` shows how the code changed over time.
- `Grading` lets you enter the score and written feedback.

## Using Replay

Replay is useful for questions like:

- how the student built the solution
- whether the work evolved gradually
- which parts of the project changed during the session
- whether the student revised and debugged over time

## Flagging a Submission

Teachers can flag a submission when something needs extra attention.

Examples:

- something unusual appeared in replay
- you want to revisit the submission later
- another instructor or TA should inspect it

## Grading a Submission

1. Open the student's review page.
2. Use replay if you need more context first.
3. Enter a grade.
4. Write feedback.
4. Click `Save Grade & Feedback`.

You can update the grade or feedback later if needed.

## Writing Good Feedback

Helpful feedback is:

- specific
- respectful
- clear
- tied to the assignment goals

Useful feedback often explains:

- what the student did well
- what is incorrect or incomplete
- what should improve next time

Saving a grade is not the same as publishing it. Publishing is the step that makes the results visible to students.

Use the assignment page to manage release:

1. Return to the assignment after grading submissions.
2. Check the `Statistics` card to see how many submissions are graded.
3. Use the publish action in that right-hand assignment sidebar when you are ready to release results.
4. After publishing, Leopard marks the assignment with `Published grades`.

Once published, students can immediately see their grade and feedback on the assignment page.

## Tips

- Use replay to add context, not as a replacement for review.
- Save grade and feedback before publishing.
- Publish grades only when you are ready for students to see them.
