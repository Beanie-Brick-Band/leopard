# Review, Grading, and Publishing

## Viewing Submissions

The assignment page includes a submissions table.

This helps you quickly see:

- who submitted
- when they submitted
- whether the submission is flagged
- whether it has already been graded

Each row includes a `Review` action.

## Reviewing a Submission

The review page is where you inspect a student's work in detail.

It usually shows:

- student name
- current grade
- linked workspace information
- replay viewer
- grade input
- feedback field

## Using Replay

Replay lets you scrub through the coding history for the submission workspace.

Replay can help you understand:

- how the student built the solution
- whether the work was gradual or sudden
- which files changed
- whether the student revised and debugged over time

Typical replay use looks like this:

1. Open a student's submission.
2. Start replay.
3. Move backward and forward through the timeline.
4. Watch how files changed.
5. Use that context while grading.

## Flagging a Submission

Teachers can flag a submission when something needs extra attention.

Examples:

- something unusual appeared in replay
- you want to revisit the submission later
- another instructor or TA should inspect it

## Grading a Submission

To grade a submission:

1. Open the student's review page.
2. Enter a grade.
3. Write feedback.
4. Click `Save Grade & Feedback`.

You can update the grade or feedback later if needed.

## Writing Good Feedback

Helpful feedback is:

- specific
- respectful
- clear
- tied to the assignment goals

Useful feedback often explains:

- what the student did well
- what is incorrect or incomplete
- what should improve next time

## Publishing Grades

Saving a grade is not the same as publishing it.

Publishing is the step that makes grades visible to students.

To publish grades:

1. Open the assignment page.
2. Review the statistics.
3. Click `Publish Grades`.
4. Confirm the action.

After publishing:

- students can immediately see their grades and feedback

## Assignment Statistics

The teacher assignment page may show:

- total students
- how many submitted
- how many were graded
- average grade
- how many grades were published

This helps you track progress during grading.

## Tips

- Use replay to add context, not as a replacement for review.
- Save grade and feedback before publishing.
- Publish grades only when you are ready for students to see them.
